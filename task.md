# Финальный проект: описание и требования

## Дедлайн

31.12.2025

Поздравляем с приближением к завершению курса! Финальный проект – это ваша возможность продемонстрировать все полученные знания и навыки в области MLOps на практике. В отличие от предыдущих заданий, где мы давали конкретную задачу, теперь вам предстоит придумать и реализовать свой собственный Pet-проект.

**Цель финального проекта** — не обязательно создать новый продукт, а построить целостную, надежную и воспроизводимую MLOps-систему, решающую понятную бизнес-задачу.

**Задача** — выберите любую понятную вам проблему (прогноз спроса, классификация текстов/изображений, детекция аномалий, рекомендательная система и т.п.), разработайте базовое ML-решение и превратите его в работающий сервис.

## Компоненты проекта

1. **Бизнес-ценность (обязательно)**
   - В README должна быть секция, чётко и кратко описывающая проблему и ценность.
   - One-pager (PDF/MD/HTML) с саммари: бизнес-задача, данные, стек, метрики/пропускная способность.
2. **Воспроизводимость и документация (обязательно)**
   - `docker-compose.yml` — одна команда для запуска всей системы.
   - README с архитектурной схемой (можно draw.io) и инструкцией по запуску.
   - `requirements.txt` (или аналоги) для всех Python-зависимостей.
3. **ML-сервис** — инференс внутри сервиса (обучение можно offline).
4. **База данных** — PostgreSQL/ClickHouse/Redis и др. для хранения данных, результатов, метаданных или фичей.
5. **Шедулер с ETL** — Airflow/Dagster/dbt для витрин, фичей или оффлайн предсказаний по расписанию.
6. **Брокер сообщений** — Kafka/RabbitMQ для асинхронной обработки (поток инференса или результатов).
7. **Простой UI** — Streamlit/Taipy/Gradio/Flask:
   - Инференс/отправка данных, просмотр результатов/метрик.
8. **Мониторинг** — дашборд в Grafana, подключенный к БД или данным приложения (запросы, распределения, метрики).

## Оценивание

### На 6–7 баллов
- Публичный GitHub-репозиторий.
- Используется новый сервис (не датасеты из домашних заданий).
- `docker-compose.yml` поднимает все обязательные сервисы (ML-сервис, БД, брокер, UI и пр.).
- README с бизнес-описанием, архитектурой и понятной инструкцией по запуску.
- Проект работает в соответствии с описанием.

### На 8–10 баллов (дополнительно, минимум 2 пункта)
1. Расширенный мониторинг/логирование (Grafana: бизнес/ML-метрики, логирование важных событий).
2. Переобучение/апдейт модели без остановки сервиса (сигнал или расписание, например Airflow).
3. Продвинутый UI: аналитика, графики, фильтры; OpenAPI/Swagger для ML-сервиса.
4. Сервис инференса в связке с брокером сообщений.
5. База данных с ETL/оркестратором: сбор метрик/фич/витрин по расписанию.

## Как отправить результат

Предоставьте ссылку на публичный GitHub-репозиторий, содержащий:
- Весь код проекта.
- Dockerfile(ы) и `docker-compose.yml`.
- Исходники one-pager (PDF/MD/HTML).
- README с инструкцией по запуску и описанием архитектуры.

Репозиторий должен запускаться одной командой `docker-compose up` без проблем. Проявите креативность и выберите задачу, которая интересна именно вам — постройте свою первую полноценную MLOps-систему. Желаем удачи!
